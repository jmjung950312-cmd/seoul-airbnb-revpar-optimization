# 포트폴리오 요약: 서울 에어비앤비 수익 최적화 대시보드

## 3줄 요약

서울 32,061개 에어비앤비 리스팅 데이터를 분석하여, 호스트가 숙소 수익을 극대화할 수 있는 **맞춤형 수익 최적화 대시보드**를 개발했습니다. LightGBM 기반 RevPAR(Revenue Per Available Room) 예측 모델과 5-컴포넌트 운영 헬스 스코어를 통해, 호스트에게 데이터 기반 요금 전략과 구체적인 운영 개선 액션을 제공합니다. 5단계 위자드 플로우로 3분 만에 숙소 수익 진단을 완료할 수 있는 직관적인 사용자 경험을 설계했습니다.

## 이력서용 프로젝트 설명

### 서울 에어비앤비 수익 최적화 대시보드 | Python, Streamlit, LightGBM

- 서울 25개 자치구 32,061개 에어비앤비 리스팅 데이터 분석 및 K-Means 클러스터링으로 4개 시장 유형 분류
- LightGBM 앙상블 모델(ADR + 예약률) + Isotonic Regression 보정으로 RevPAR 예측 파이프라인 구축
- Streamlit 기반 5단계 위자드 대시보드 개발 (수익 요약, 요금 추천, 관광지 분석, 지역 진단, 헬스 스코어, 숙소 설명 6개 탭)

## 기술적 의사결정

### 왜 LightGBM 2-모델 앙상블인가?

RevPAR = ADR x 예약률이라는 도메인 구조를 반영하여, ADR 예측(Model A)과 예약률 예측(Model B)을 분리했습니다. 각 모델이 서로 다른 피처 세트를 사용하므로 해석 가능성이 높고, price_gap(호스트 실제 요금과 시장 적정 요금의 차이)을 Model B에 피딩하여 가격 탄력성까지 반영했습니다.

### 왜 Isotonic Regression 보정인가?

LightGBM의 raw 예측값은 극단값에서 과대/과소 추정이 발생할 수 있습니다. Isotonic Regression으로 단조 증가 보정을 적용하여 예측 신뢰도를 개선했습니다.

### 왜 단일 파일(app.py) 구조인가?

Streamlit Cloud 배포 환경에서의 단순성과, 위자드 플로우의 session_state 관리를 고려하여 단일 파일 구조를 선택했습니다. CSS 변수 기반 테마 시스템과 명확한 영역 분리(CSS → 상수 → ML → 헬퍼 → UI → Step 함수)로 유지보수성을 확보했습니다. 향후 사용자 규모 확장 시 Step 함수별 모듈 분리 및 라우팅 도입을 계획하고 있습니다.
