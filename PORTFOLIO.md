# 포트폴리오 요약: 서울 에어비앤비 호스트 수익 최적화 분석

## 3줄 요약

서울 32,061개 에어비앤비 리스팅 데이터를 분석하여, 호스트 관점에서 RevPAR을 최적화할 수 있는 핵심 변수와 전략을 도출했습니다. LightGBM 2-Stage 파이프라인(ADR + 예약률 분리 예측)과 K-Means 클러스터링으로 4개 시장 유형별 차별화 전략을 제시하며, Streamlit 대시보드로 3분 만에 숙소별 맞춤 수익 진단을 제공합니다. 분석 노트북부터 ML 모델, 인터랙티브 대시보드까지 End-to-End 데이터 분석 프로젝트입니다.

## 기술적 의사결정

### 왜 LightGBM 2-모델 앙상블인가?

RevPAR = ADR x 예약률이라는 도메인 구조를 반영하여, ADR 예측(Model A)과 예약률 예측(Model B)을 분리했습니다. 각 모델이 서로 다른 피처 세트를 사용하므로 해석 가능성이 높고, price_gap(호스트 실제 요금과 시장 적정 요금의 차이)을 Model B에 피딩하여 가격 탄력성까지 반영했습니다.

### 왜 Isotonic Regression 보정인가?

LightGBM의 raw 예측값은 극단값에서 과대/과소 추정이 발생할 수 있습니다. Isotonic Regression으로 단조 증가 보정을 적용하여 예측 신뢰도를 개선했습니다.

### 왜 K-Means k=4인가?

실루엣 분석에서 k=4가 최적 분리도를 보였습니다. 4개 클러스터(핫플 수익형, 프리미엄 비즈니스, 로컬 주거형, 가성비 신흥형)는 자치구별 RevPAR 수준, 공급 밀도, Dormant 비율 등 핵심 변수에서 뚜렷한 차이를 보입니다.

### 왜 단일 파일(app.py) 구조인가?

Streamlit Cloud 배포 환경에서의 단순성과, 위자드 플로우의 session_state 관리를 고려하여 단일 파일 구조를 선택했습니다. CSS 변수 기반 테마 시스템과 명확한 영역 분리(CSS → 상수 → ML → 헬퍼 → UI → Step 함수)로 유지보수성을 확보했습니다. 향후 사용자 규모 확장 시 Step 함수별 모듈 분리 및 라우팅 도입을 계획하고 있습니다.

### 왜 OOF(Out-of-Fold)인가?

Model A의 ADR 예측값과 호스트 실제 ADR의 차이(price_gap)를 Model B의 입력으로 사용합니다. 학습 데이터에서 직접 예측하면 과적합 정보가 누수되므로, 5-Fold OOF 방식으로 price_gap을 생성하여 누수를 차단했습니다.

## 주요 도메인 인사이트

- **슈퍼호스트 프리미엄 +83.1%**: Active+Operating 기준, 슈퍼호스트 RevPAR ₩61,205 vs 일반 ₩31,825
- **Dormant 비율 54.3%**: 전체 리스팅의 절반 이상이 비활성 — 시장 건전성 핵심 리스크
- **사진 최적 구간 21~35장**: 사진 수 증가에 따른 RevPAR 상승이 35장 이후 둔화
- **최소숙박일 최적점 2~3박**: 예약 가능일 극대화와 리뷰 축적 속도의 균형점
- **마포구 공급 집중 (~21%)**: 과잉 공급으로 인한 RevPAR 압박 구조
